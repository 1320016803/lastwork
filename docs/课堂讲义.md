# 课堂讲义：用模板发布你的第一个 Python 桌面小应用

> 面向：**零基础、非计算机专业**、Python 入门课同学。  
> 本课配套仓库：**python-desktop-app-template**（见仓库根目录 [README](../README.md)）。

---

## 课时建议拆分（教师可按机房课时调整）

| 课时 | 主题 | 学生动手 / 智能体 |
|------|------|-------------------|
| 第 1 节 | Gitee 是什么、从模板得到自己的仓库、clone 到本机 | 人类 `git clone` 或复制 README「安装」指挥句给 WorkBuddy |
| 第 2 节 | 建 venv、装依赖、跑通 `main.py` | 按 README「人类读者：怎么运行」操作 |

| 第 4 节 | 可选：换图标；改 README 占位链接与快捷方式名 | 人类编辑 + 可选请 WorkBuddy 跑脚本 |
| 第 5 节 | `git` 提交与推送、仓库设公开、同桌互测 | 推送密码/令牌由学生本人在系统里输入，不发聊天 |

## 本课你要学会什么（用大白话）

1. 知道 **Gitee** 上可以存自己的「小软件文件夹」，别人能下载运行。  
2. 会 **改模板里的窗口和按钮**，变成你自己的小应用。  
3. 会 **让 WorkBuddy（智能体）帮你** 完成很多「敲命令」的步骤，你只要会 **复制、粘贴、点允许**。  
4. 知道 **不要把密码、令牌发到聊天里**；需要登录时只在系统弹窗里自己输入。

## 课前准备（你或机房老师装好即可）

- **Python 3.10 或以上**（安装时勾选 **pip**）。  
- **Git**（用来「下载/更新」Gitee 上的项目）。  
- **WorkBuddy**（或老师指定的智能体），能执行终端命令。  
- 一个 **Gitee 账号**（用来建你自己的仓库、上传代码）。

## 第一步：从模板得到你自己的仓库

老师会在 Gitee 上准备一个 **模板仓库**（名字可能是 `python-desktop-app-template`）。

你需要做的是（任选一种，**以你学校 Gitee 页面上能点的为准**）：

- **方式 A**：新建仓库时选 **「使用模板」**，选中老师的模板，起一个新名字（例如 `我的计算器`）。  
- **方式 B**：在模板仓库页面点 **Fork**，再改仓库名（如有需要）。  
- **方式 C**：没有模板功能时，按老师发的图文：新建空仓库，再把模板压缩包解压后上传（老师会给步骤）。

**重要**：记下你仓库的 **HTTPS 地址**，形如：  
`https://gitee.com/你的用户名/你的仓库名`

后面 README 里所有 **`<你的用户名>`**、**`<你的仓库名>`** 都要换成这个。

---

## 第二步：把项目弄到本机（两种办法）

### 办法 1：自己敲（练一练终端）

1. 打开终端（Windows 搜 **PowerShell**）。  
2. 执行（换成你的地址；**最后一段 `cd` 进你的仓库名**）：

```bash
git clone https://gitee.com/你的用户名/你的仓库名.git
cd 你的仓库名
```

### 办法 2：交给 WorkBuddy（推荐）

打开 README，找到 **「复制给 WorkBuddy」** 里 **安装** 那一段，  
把里面的 `<你的用户名>`、`<你的仓库名>` 先改好，**整段复制**发给 WorkBuddy。  
它做完后，问清楚：**项目在你电脑哪个文件夹里**。

---

## 第三步：先跑起来模板

在 **项目文件夹里** 打开终端（或先 `cd` 进去），按 README **「人类读者：怎么运行」** 安装依赖，再：

```bash
python main.py
```

看到窗口里能 **点按钮计数**，就说明环境 OK。

**常见卡点**：  
- 提示 **找不到 python**：Windows 试试 `py -3 main.py`。  
- **装包很慢**：加清华镜像（README 里有示例）。  
- 提示 **不能全局 pip**：按 README 建 **`.venv`**，再用里面的 `python`。

---

## 第四步：改成你自己的程序

1. 打开 **`main.py`**，找到顶上的 **`APP_TITLE`**，改成你的应用名字。  
2. 改界面、加按钮、写你自己的逻辑（本课以老师要求为准）。  
3. 若用了新依赖，写进 **`requirements.txt`**（一行一个包名）。

保存后再次 `python main.py` 自测。

---

## 第五步：图标（可选但好看）

1. 用豆包等工具生成一张 **正方形** 小图（像素风、可爱均可）。  
2. 在 Mac/Linux 若已装 **ImageMagick**，可在项目里执行：

```bash
bash scripts/build_app_ico.sh 你的图.png
```

会生成 **`assets/app.ico`**。Windows 可请 WorkBuddy 或老师帮跑，或用在线转 ico 工具后覆盖 `assets/app.ico`。  
3. 程序已自动加载 `assets/app.ico`，换文件即可换图标。

---

## 第六步：改 README（给使用你软件的人）

打开根目录 **README.md**，建议至少改这些：

- 顶部的 **标题、简介**（别再用「模板」字样）。  
- 所有 **`https://gitee.com/<你的用户名>/<你的仓库名>`** 换成你**真实**的地址。  
- 若本地文件夹名不是 `python-desktop-app-template`，把文中路径 **全局替换成你的文件夹名/仓库名**（含智能体剧本里的 `$Repo` / `REPO`）。  
- **「复制给 WorkBuddy」** 两段话里的链接也要改。  
- 桌面快捷方式名字（默认 **「我的桌面应用」**）可改成你的应用名，并全文替换 **`.lnk` 相关说明** 里的文件名。

**对照学习**：老师示例 [caesar-desktop-lab](https://gitee.com/thiswind/caesar-desktop-lab) 的 README 结构和本模板一致，可打开对照改。

---

## 第七步：上传（发布）到 Gitee

**你要明白**：别人 **下载（clone）你的公开仓库** 一般 **不要密码**；  
但 **你往自己仓库传代码（push）** 需要证明你是本人——通常用 **私人令牌（PAT）** 或 **SSH**，**不要**把令牌发到聊天里。

**交给 WorkBuddy 的示例话术**（在项目文件夹里操作，已 `git` 初始化过则跳过 `init`）：

```text
请在本机项目目录执行：git status 查看修改；然后 git add . ，git commit -m "描述你的修改"，再推送到 origin。推送时如果需要密码，请提示用户本人在弹窗中输入 Gitee 私人令牌，不要把令牌写在文件里。
```

（具体 `remote` 地址以你 Gitee 网页上的 HTTPS 为准；老师可上课演示一次。）

最后在 Gitee 网页：**管理 → 基本设置 → 是否开源**，设为 **公开**，同学才能免密克隆。

---

## 第八步：验收（同桌互测）

1. 把 README 里 **「安装请发送」** 那段（已改成你的仓库地址）发给同桌的 WorkBuddy。  
2. 同桌能 **克隆并运行** 你的程序，即通过。

## FAQ（生活化）

**问：我根本没学过「终端」，会不会挂科？**  
答：本课不要求你背命令。你会 **复制 README 里的话**、会 **看窗口里有没有报错** 就行；复杂步骤可以交给 WorkBuddy，但你要能说出「项目在哪个文件夹」。

**问：WorkBuddy 要我输入 Gitee 密码，我该发在聊天里吗？**  
答：**不要。** 密码、私人令牌只在 **浏览器或系统弹窗** 里自己输入。智能体只应提示你去哪里点、去哪里填。

**问：同桌克隆我的仓库还要登录？**  
答：多半是你的仓库还是 **私有**。去 Gitee：**管理 → 基本设置 → 是否开源 → 公开**。

**问：我改了很多代码，一运行就闪退怎么办？**  
答：先用终端运行 `python main.py`（不要双击），**看黑色窗口里的英文报错**；把最后几行抄给老师或 WorkBuddy（不要抄含密码的内容）。

**问：文件夹名和仓库名不一样要紧吗？**  
答：不要紧，但 README 和智能体剧本里的路径要跟着 **你本机实际文件夹名** 改，否则快捷方式会指错地方。

**问：卸载会不会把我电脑里别的 Python 项目删掉？**  
答：按 README **卸载剧本** 只会删 **约定好的那一个文件夹** 和 **约定好的快捷方式**。删之前看清楚路径，不确定就问老师。

## 可复制给 WorkBuddy 的长话术汇总

以下与根目录 [README](../README.md) 中「复制给 WorkBuddy」及推送示例一致，便于讲义打印或投屏时 **一页集中展示**（使用前仍须替换 `<你的用户名>`、`<你的仓库名>`）。

**安装（克隆 + 依赖 + 可选快捷方式）**

```text
请打开（或克隆）公开仓库 https://gitee.com/<你的用户名>/<你的仓库名> ，阅读 README 里的「给智能体的安装剧本」，根据我当前操作系统逐步执行。克隆公开仓库时不要向用户索要 Gitee 账号或密码。若某一步失败，按 README 里的说明降级处理（尤其是桌面快捷方式）。最后请明确告诉我：项目目录路径、用哪条命令启动（若用了 venv，要写全 venv 里的 python/pythonw 路径）。若 README 里写 python-desktop-app-template，而我的仓库名不同，请用实际文件夹名替换路径。
```

**卸载**

```text
请阅读仓库 https://gitee.com/<你的用户名>/<你的仓库名> 的 README 中「给智能体的卸载剧本」，在我当前操作系统下按步骤执行，并在完成后告诉我删除了哪些路径。若本地文件夹名与仓库名不一致，请按实际文件夹名删除。
```

**推送自己的修改（需本人令牌时）**

```text
请在本机项目目录执行：git status 查看修改；然后 git add . ，git commit -m "描述你的修改"，再推送到 origin。推送时如果需要密码，请提示用户本人在弹窗中输入 Gitee 私人令牌，不要把令牌写在文件里。
```

## 附录：智能体不能替你做的事

- 在浏览器里点「公开仓库」「生成令牌」等按钮。  
- 在系统密码框里输入你的账号、令牌。  
- 替你决定「删不删整个文件夹」（卸载前要你自己确认）。

## Gitee 模板与备选流程（教师课前核对）

以下用于 **在目标 Gitee 环境完成一次验证**，并把结果写进第一次课的板书或教研记录。不要求学生逐条操作。

### 1. 核对「模板仓库」功能是否可用

1. 用 **教师账号** 登录 Gitee。  
2. 打开（或先新建）源仓库 **python-desktop-app-template**。  
3. 进入 **管理 → 功能设置**（或「仓库设置」中同类入口，以当前 Gitee 界面为准）。  
4. 查找 **「模板仓库」** / **「作为模板」** 类开关：  
   - **若存在且可开启**：开启后保存。用 **另一测试账号** 或无痕窗口验证：**新建仓库 → 使用模板** 能否选中本仓库并生成新仓库。  
   - **若不存在或灰显**：记录界面文案与版本，改走下面 **备选 A / B**。

### 2. 备选 A：Fork

1. 在模板仓库页点击 **Fork** 到自己的命名空间。  
2. 将 Fork 得到的仓库 **改为公开**（若默认私有）。  
3. 确认 **只读 clone** 不需要密码：`git clone https://gitee.com/<教师测试用户名>/<fork后的仓库名>.git`。

### 3. 备选 B：空仓库 + 上传 / 导入

1. 新建 **空仓库**（公开）。  
2. 任选：本机 `git push`，或使用 Gitee **「导入仓库」** 从已有 URL 导入模板。  
3. 确认学生拿到的 **HTTPS 地址** 与讲义中的替换说明一致。

### 4. 写入教学材料时的建议表述

- 在 **第一次课** 明确告诉学生：「我们班使用 **方式 A / B / C** 得到自己的仓库」，并演示一次。  
- 在 README 顶部的课堂链接旁，可注明「以教师课堂说明为准」。
